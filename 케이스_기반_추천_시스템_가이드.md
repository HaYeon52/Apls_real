# 📚 케이스 기반 과목 추천 시스템 사용 가이드

## 🎯 개요

기존의 가중치 기반 추천 알고리즘을 **완전히 제거**하고, 관심분야 선택에 따른 **16가지 케이스별 하드코딩 추천 시스템**으로 변경되었습니다.

## 📋 16가지 케이스 목록

### 단일 관심분야 (6개)
1. **관심 없음** - `no-interest`
2. **공정 (생산, 품질)** - `manufacturing`
3. **물류/SCM** - `logistics`
4. **데이터** - `data`
5. **금융** - `finance`
6. **컨설팅/기획** - `consulting`

### 조합 관심분야 (10개)
7. **공정 + 물류** - `manufacturing-logistics`
8. **공정 + 데이터** - `manufacturing-data`
9. **공정 + 금융** - `manufacturing-finance`
10. **공정 + 컨설팅** - `manufacturing-consulting`
11. **물류 + 데이터** - `logistics-data`
12. **물류 + 금융** - `logistics-finance`
13. **물류 + 컨설팅** - `logistics-consulting`
14. **데이터 + 금융** - `data-finance`
15. **데이터 + 컨설팅** - `data-consulting`
16. **금융 + 컨설팅** - `finance-consulting`

## 📝 데이터 입력 방법

### 1. 파일 위치
`/src/app/utils/caseBasedRecommendations.ts`

### 2. 데이터 구조

각 케이스별로 다음 정보를 입력합니다:

```typescript
{
  caseId: "케이스_아이디",
  caseName: "케이스 이름",
  description: "케이스 설명",
  semesters: [
    {
      semester: "1-2", // 학기 (1-1, 1-2, 2-1, 2-2, 3-1, 3-2, 4-1)
      courses: [
        {
          courseName: "과목명", // courseData.ts의 name과 정확히 일치해야 함
          reason: "추천 이유"
        }
      ]
    }
  ]
}
```

### 3. 예시: "관심 없음" 케이스

```typescript
"no-interest": {
  caseId: "no-interest",
  caseName: "관심 없음 (진로 탐색)",
  description: "진로를 아직 결정하지 못한 학생을 위한 탐색형 추천입니다.",
  semesters: [
    {
      semester: "1-2",
      courses: [
        {
          courseName: "산업공학개론",
          reason: "산업공학의 전체 분야를 개괄적으로 배우며 진로 방향을 탐색할 수 있습니다."
        },
        {
          courseName: "산업인공지능시스템응용",
          reason: "데이터와 AI의 기초를 경험하며 자신의 적성을 파악할 수 있습니다."
        }
      ]
    },
    {
      semester: "2-1",
      courses: [
        {
          courseName: "산공수학",
          reason: "모든 전공 분야의 기초가 되는 수학적 도구를 배웁니다."
        }
      ]
    }
    // ... 나머지 학기
  ]
}
```

## ⚠️ 주의사항

### 1. 과목명 정확성
- `courseName`은 `/src/app/utils/courseData.ts`의 과목명과 **정확히 일치**해야 합니다
- 띄어쓰기나 특수문자도 정확히 일치해야 합니다
- 예: `"산업공학개론"` (O) vs `"산업 공학 개론"` (X)

### 2. 학기 형식
- 반드시 `"1-1"`, `"1-2"`, `"2-1"`, `"2-2"`, `"3-1"`, `"3-2"`, `"4-1"` 형식 사용
- 4학년 2학기는 추천하지 않음 (졸업 학기)

### 3. 필수 과목
- **전공기초(필수)** 과목은 자동으로 추가되므로 입력하지 않아도 됩니다
- 선택 과목만 입력하세요

### 4. 추천 개수
- 각 학기당 **최대 3개의 선택 과목** 추천 권장
- 너무 많으면 사용자가 선택하기 어려움

## 🔄 데이터 입력 워크플로우

### 단계 1: 케이스 선택
16개 케이스 중 입력할 케이스를 선택합니다.

### 단계 2: 학기별 과목 정리
각 학기(1-2 ~ 4-1)에 추천할 선택 과목을 정리합니다.

### 단계 3: 추천 이유 작성
각 과목마다 해당 케이스에 맞는 추천 이유를 작성합니다.

### 단계 4: 코드에 입력
`caseBasedRecommendations.ts` 파일에서 해당 케이스의 `semesters` 배열에 데이터를 입력합니다.

### 단계 5: 과목명 검증
입력한 과목명이 `courseData.ts`에 존재하는지 확인합니다.

## 📊 시스템 동작 방식

1. 사용자가 관심분야를 선택 (최대 2개)
2. `getCaseId()` 함수가 케이스 ID를 결정
3. `caseBasedRecommendations`에서 해당 케이스 데이터 로드
4. 사용자의 현재 학기와 수강 완료 과목을 고려하여 필터링
5. 필수 과목 + 케이스별 추천 선택 과목을 결합하여 표시
6. 각 과목에 케이스별 추천 이유(`reason`) 표시

## 🎨 화면 표시

추천 결과 화면에서:
- 💡 **추천 근거** 섹션에 `reason` 필드의 내용이 표시됩니다
- 케이스별 맞춤 추천 이유가 파란색 박스에 강조 표시됩니다
- 필수 과목은 "필수 과목입니다. 반드시 수강해야 합니다."로 자동 표시됩니다

## 🛠️ 데이터 입력 팁

### 추천 이유 작성 가이드
- **구체적으로**: "왜 이 과목을 이 케이스에 추천하는지" 명확히 설명
- **진로 연계**: 선택한 관심분야와 과목의 연관성 강조
- **실무 활용**: 실제 진로에서 어떻게 활용되는지 설명
- **학습 로드맵**: 이전/이후 과목과의 연계성 설명

### 예시: 좋은 추천 이유
```typescript
{
  courseName: "데이터마이닝과기계학습",
  reason: "제조 공정 데이터를 분석하여 불량률을 예측하고 품질을 개선하는 실무 역량을 기릅니다. 스마트팩토리개론에서 배운 제조 시스템 지식과 결합하여 데이터 기반 의사결정 능력을 강화할 수 있습니다."
}
```

### 예시: 피해야 할 추천 이유
```typescript
{
  courseName: "데이터마이닝과기계학습",
  reason: "데이터를 배웁니다." // ❌ 너무 추상적
}
```

## 🔍 디버깅

### 콘솔 로그 확인
브라우저 개발자 도구의 콘솔에서 다음 메시지를 확인할 수 있습니다:
- `🎯 [케이스 기반 추천 시스템 시작]`
- `✅ 적용된 케이스: [케이스명] ([케이스ID])`
- `📋 [현재 학기 추천] X-X: N개 과목`
- `✅ [학기별 추천 완료] 총 N개 학기`

### 일반적인 오류
1. **과목을 찾을 수 없습니다**: `courseName`이 `courseData.ts`와 일치하지 않음
2. **추천 과목이 없음**: 해당 케이스의 데이터가 아직 입력되지 않음
3. **케이스 데이터를 찾을 수 없습니다**: `caseId` 매핑이 잘못됨
